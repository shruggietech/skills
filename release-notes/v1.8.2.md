# v1.8.2 - 2026-07-19

## Changed

- `shruggie-markdown`: the line-length rule now defines a full wrapping standard. A single
  `Line length and wrapping` subsection sets the 80-column house default, recognizes the `{100,
  120}` overrides in plain language, and adds an opt-in "no hard wrap" soft-wrapping mode. It
  enumerates the wrap exemptions (headings, fenced code, GFM tables, both front-matter forms,
  reference and data-URI definitions, unbreakable tokens) and the complete set of forbidden
  continuation-line leading markers, with the "safety over width" break rule so a wrapped list
  continuation is never misparsed as a nested bullet or ordered list. `authoring-reference.md`
  gains worked examples for the spaced-hyphen and year (`2016.`) wrap traps, and the frontmatter
  triggers on "wrap this at 80" and "fix the line length".
